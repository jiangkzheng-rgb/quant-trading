# 量化库版加仓策略说明

更新日期：2026-06-16

## 目标

把原来的“固定回撤阈值加仓”升级为“历史分位 + 趋势过滤 + 波动控制 + 资金约束”的规则。

它仍然只做一件事：在长期只买不卖的前提下，判断今天是否只是执行基础定投，还是允许使用一部分额外预算。

## 工具分工

| 工具 | 用途 |
|---|---|
| yfinance | 下载 Nasdaq-100、黄金期货、ETF 等历史日线 |
| vectorbt | 做向量化回测，比较等额定投、回撤加仓、均线过滤策略 |
| QuantStats | 输出收益、回撤、波动率、夏普、月度表现等报告 |
| OpenBB | 后续接入宏观利率、美元指数、黄金驱动因素 |
| edgartools / sec-edgar-downloader | 后续分析美股科技龙头财报、行业基本面 |

| FinRL | 后续测试强化学习仓位管理，但不作为默认实盘规则 |

默认产品策略不使用黑箱模型，仍使用可解释规则。

## 新策略核心

### 1. 历史回撤分位

不再只看“跌了多少”，而是看当前回撤在历史里有多极端。

- 纳指使用最近 756 个交易日，约 3 年
- 黄金使用最近 252 个交易日，约 1 年
- 回撤分位越高，说明当前位置越接近历史深跌区

### 2. 趋势过滤

使用 200 日均线控制追高风险。

- 低于 200 日均线：额外预算分提高
- 高于 200 日均线 12%-18% 且回撤很浅：暂停额外预算
- 200 日均线斜率转弱但回撤不够深：等待

### 3. 过热/超卖过滤

使用 RSI 14 和 60 日年化波动。

- RSI 低于 30：短期超卖，允许小幅加分
- RSI 高于 72：短期偏热，降低额外预算
- 波动率过高：不取消预算，但降低力度，拆分执行

### 4. 资金约束

金额必须受以下条件限制：

- 用户设置的可用预算
- 备用金比例
- 单次额外上限
- 今日已额外记录金额
- 基础定投金额

## 纳指100默认规则

| 参考值 | 状态 | 处理 |
---|---|---|
| 0-41 | 日常区 | 只做基础定投 |
| 42-55 | 观察区 | 不额外使用预算 |
| 56-63 | 轻量区 | 小额额外预算 |
| 64-75 | 稳健区 | 分批使用预算 |
| 76-87 | 深回撤区 | 提高额外预算 |
| 88-100 | 历史深回撤区 | 可用更高额度，但仍受单次上限控制 |

## 黄金默认规则

黄金的额外预算更保守，因为历史回测显示黄金回撤加仓相对等额定投的收益优势不稳定。

| 参考值 | 状态 | 处理 |
---|---|---|
| 0-47 | 日常区 | 只做基础记录 |
| 48-61 | 观察区 | 不额外使用预算 |
| 62-69 | 轻量区 | 小额预算记录 |
| 70-81 | 偏低区 | 稳健预算记录 |
| 82-100 | 深回撤区 | 允许额外预算，但单笔严格封顶 |

## 重要结论

1. 纳指100：历史分位回撤策略比固定阈值更稳，因为它能适配不同波动阶段。
2. 黄金：适合小额、慢节奏、严格限额，不能照搬纳指的进攻规则。
3. 一次性买入长期期望可能更高，但路径回撤更大，不适合容易中途放弃的人。
4. 等额定投仍是基准，额外预算只在统计条件足够好时使用。
5. 回测不是保证收益，只是减少情绪化操作。
